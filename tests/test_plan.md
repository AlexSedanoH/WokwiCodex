# Checklist mínimo de pruebas y entrega

## 1) Checklist de pruebas mínimas
- [ ] Ejecuté el flujo principal sin error.
- [ ] Probé al menos una entrada válida.
- [ ] Probé al menos una entrada inválida.
- [ ] Probé un caso límite pequeño.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Completé `docs/plan_de_pruebas.md`.
- [ ] Mi alcance y criterios de éxito están claros.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` detecta el archivo principal.
- [ ] El comando principal de mi proyecto está documentado.
- [ ] No uso dependencias complejas ni servicios externos.

## 4) Checklist de entrega final
- [ ] El repositorio mantiene la estructura pedida.
- [ ] Mi proyecto sigue siendo pequeño y viable.
- [ ] La documentación está primero; el prototipo es opcional y mínimo.
